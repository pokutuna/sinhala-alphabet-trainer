# 準備メモ: シンハラ文字学習アプリ

設計の出発点となった検討内容のまとめ。元になった設計会話: https://claude.ai/share/55fdd95f-abc4-4e8c-b0e7-8ae897f66191

## このアプリは何か

**シンハラ「文字」**(書記体系)を覚えるためのインタラクティブな学習 Web アプリ。

- 「シンハラ語」は言語、「シンハラ文字 (Sinhala script)」は文字体系。学ぶ対象は後者。
- 文字種は **アブギダ**(子音字が母音 a を内蔵する音節文字)。アルファベットではない。
- 字母表 (hōdiya) は数え方で字数が変わる: フルセット(混成字母 miśra)約54字、実用コア(純粋字母 śuddha)約38字、高頻度に絞ると30字弱。

## 学習対象の構造 = 「4つの部品」モデル

シンハラ文字は次の部品の掛け算で組み立てる。総当たり暗記ではなく足し算で量が減る、というのが学習方針の核。

| 部品 | 役割 | 例 | JSON のキー |
| --- | --- | --- | --- |
| ① 子音字 | 単体で「子音+a」を表す基本字 | ක = ka | `consonants` |
| ② 独立母音字 | 語頭で母音が来るときのフル字 | අ = a | `independent_vowels` |
| ③ 母音記号 pilla | 子音に付けて a を別母音に変える印 (13種) | කි = k+i | `vowel_signs` |
| ④ hal kirīma | 母音を消して子音単独にする印 ් | ක් = k | `hal` |

組み立ての式:

```
語頭の母音      → ② 独立母音字をそのまま置く
子音 + 母音     → ① 子音字 + ③ pilla
子音だけ(連続)  → ① 子音字 + ④ hal (◌්)
```

## 子音は 2 軸マトリクス(五十音図的)

子音はフラットな一覧ではなく **調音点 (place) × 調音法 (manner)** の格子で整理すると、日本語の濁音・半濁音(か→が、は→ば→ぱ)のような親戚関係が見えて覚えやすい。

- 横移動 = 「声を足す/息を足す/鼻に抜く」操作
- 純粋シンハラ (native 音) に必要なのは実質「無声無気・有声無気・鼻音・前鼻音化」の列。**有気音 (kha, gha…) や śa/ṣa は miśra 字母** = サンスクリット/パーリ/外来語用で後回しでよい。
- 前鼻音化 4 字 (ඟ ඬ ඳ ඹ) はシンハラ特有で頻出。

このため JSON の子音は `place` / `manner` / `misra` フラグを持ち、フィルタ・並べ替えできる。

## 例外字形

ほぼ全セルは「i は上・u は下・e は左・o は両側」の規則どおりだが、**u/ū (paa pilla) だけは子音の下端に合わせて字形が崩れる**。

- 最重要: ර ru / රූ rū — 規則から推測できない独自グリフ。個別暗記。
- තු / තූ、ළු / ළූ なども連結形 (touching form) になる。

→ JSON では `irregular_forms` に分離。合成表では該当セルにフラグを立てて強調表示する想定。

## アプリの機能要件

技術的詳細には踏み込まず、SPA で次の 2 機能があればよい。

### 1. 表示モード切替(表ビュー)

- いくつかのモードをボタンで切り替え(子音のみ / 母音のみ / 変化型 など) → JSON `display_modes` の 6 種
  - 子音マトリクス(純粋のみ / 全部)、独立母音、母音記号 pilla、合成表(子音×母音)、例外字形
- 文字を大きく見やすく、**パーツごとに色分け**(base 子音 / vowel_sign / hal / conjunct) → JSON `parts_coloring`
- **複数フォントでの見え方の違い**を確認(切替/並列表示) → JSON `fonts`(Google Fonts 5 種)

### 2. クイズモード

- 「この文字は?」に対して読みを答える → JSON `quiz.modes` の 4 種
  - glyph→rom / glyph→kana / glyph→ipa / rom→glyph
- ひらがな読み・アルファベット・発音記号の**選択肢から回答**(4 択)
- distractor(誤答選択肢)は同じ pool からランダム抽出。近縁(同 place / 同 pilla 長短)を優先すると難度 UP

## データファイル

`app/data/sinhala.json` (約 19KB)。Unicode から正確に組み立て済み・JSON 検証済み。

| キー | 内容 | アプリでの用途 |
| --- | --- | --- |
| `meta` | 文字体系の前提・注意書き | 説明表示 |
| `consonants` | 子音 40 字(純粋 24 + 混成 16)。`glyph/rom/ipa/kana/place/manner/misra/note` | マトリクス表・クイズ出題プール |
| `independent_vowels` | 独立母音 15 字(短長ペア・misra フラグ) | 「母音のみ」モード |
| `vowel_signs` | 母音記号 pilla 13 種。`sign`(結合コードポイント)・`position`(left/right/top/bottom/both) | 合成生成・パーツ色分け |
| `hal` | 母音消去記号 ් | 子音単独形の生成 |
| `irregular_forms` | රු තු කු など規則外の連結字形 8 件 | 「例外」モード・強調表示 |
| `conjuncts` | yansaya / rakaransaya / repaya(上級) | 連結モード |
| `parts_coloring` | base/vowel_sign/hal/conjunct の役割・推奨色・描画注意 | パーツ色分け実装の指針 |
| `fonts` | Google Fonts 5 種 + 比較のコツ | フォント切替・並列表示 |
| `quiz` | 4 つの出題モード・選択肢生成戦略 | クイズモード |
| `display_modes` | 6 つの表示モード定義 | モード切替ボタン |

### 実装時の注意

- **パーツ色分けはコードポイント単位**で span 分割する。グラフェムクラスタ内で base + combining sign の順に並ぶため。特に **o/ō は left + right の 2 コードポイント**に分かれる。
- 合成表は `consonants(misra=false) × vowel_signs` の二重ループで `glyph + sign`、`irregular_forms` にあれば字形を差し替え・強調。
- `kana` は近似(正確な対応ではない)。クイズで誤答誘導にならないよう扱う。
